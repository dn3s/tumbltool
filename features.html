Features Supported

This is a list of TumblTool's support status of various Tumblr features. While certain features are missing, they fail gracefully by just not rendering. As such, you can use TumblTool to develop most of your theme and then use Tumblr's "workflow" for the last bit.

  • Global Blog Attributes

    This includes things like {Title} and {Description}

  • Users

    Usernames, user URLs, user titles, portraits, etc.

    These are all available for things like Group Blogs, Reblogs, Submitted Posts, Following, etc.

  • Like and Reblog Buttons

    These are provided, but this may change if Tumblr doesn't like my use of their SVGs for it. In which case I will need to figure out another solution. Either I will provide an "install script" that grabs the SVGs from their site that the user runs, or I'll make my own buttons. Or both.

  • Text Posts

  • Photo Posts

  • Video Posts

    To keep things lightweight, a "dummy" video player is used (an <iframe> with the word "VIDEO" in it)

  • Audio Posts

    To keep things lightweight, a "dummy" audio player is used (an <iframe> with the word "AUDIO" in it)

  • Chat Posts

  • Post Dates

    This is really slick. In the data file, all you provide is a UNIX-style timestamp and it computes any needed date values.

  • Link Posts

    Note that the {Target} attribute is computed, but will return blank because an option has not yet been added to the CLI to specify its behavior

  • Quote Posts

  • Answer Posts

  • Tags

    Everything Tag related (except for tag pages and {TagsAsClasses})

  • Localization

    "Fake" support is provided. Pretends language is always English. Will just print out whatever string it is you want localized, whether or not it is in the dictionary

  • Reblogs

    Works completely except for including reblog status in {TagsAsClasses}

  • TagsAsClasses

    Works completely except it doesn't yet include post sources as a tag, or reblog status

  • Pagination

    "next" and "prev" pages are supported, and have corresponding CLI options, but nothing else is supported at the moment, including "jump pagination"

  • Post IDs and Offsets

    Things like {block:Post[1-15]} or {PostID} don't work, but {block:Even} and {block:Odd} do.

  • Content Sources

    Basically anything under Post Sources

  • Panorama Posts

  • Photoset Posts

  • Notes and "Reblog Chains"

    Tumblr spits out some sort of big messy <ul> for this which will take some work to replicate accurately.

    However, {NoteCount} and {NoteCountWithLabel} work. (WAIT, DO THEY? I CAN'T REMEMBER)

  • Auto-Generated Summaries

    {PostSummary}

  • Non-Index Pages

    Search pages, Tag pages, Date pages, etc. This includes custom pages, which is super not supported.

  • Custom Variables

    Tumblr allows you to define custom variables using meta tags in your theme's head, for custom colors, images, booleans, etc.
