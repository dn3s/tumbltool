Usage

  1. Install TumblTool (obviously)

  2. Create A Directory For Your Theme

    If you're just going to be previewing your theme locally, put this directory wherever it's convenient for you, just like any other project.

    If you have set up a web server and want to preview your theme on one or more separate machines, you'll probably want your directory somewhere your web server can access (like under your DocumentRoot).

  3. Work On Your Theme File

    Hack away!

  4. Preview Your Theme

    Run tumbltool for a detailed list of options. For starters, try this: tumbltool preview -theme your_theme_file -pipe | xargs firefox

    This will render your theme and pipe it as a data:URI into firefox.

    To help keep things tidy, you don't need to write all of your code in one theme file. You can write scripts and stylesheets in external files, and have TumblTool take care of including them. -include will include files at the end of your <head>, and -tail-include will include files at the end of your <body>.

    Warning!

    Especially if you're doing anything complicated, like previewing with a web server or emailing previews to your client, you'll want to think about how TumblTool includes external files in your previews. TumblTool can either reference the files using <link> and <script> tags, or read in the files, wrap them in the appropriate tags, and insert them, inline, directly into the output. Inlining is handy because you don't have to worry about "breaking" relative URLs by moving the file around, or viewing it on another machine. You can specify this behavior with -inline-css and -inline-js (these can be negated as -no-inline-css and -no-inline-js). Aside from simplicity, inlining scripts and stylesheets with TumblTool also has the benefit of optionally minifying your code, with -minify-js and -minify-css.

    TumblTool has a stash of images for things like portraits, photo posts, etc, as well as some HTML files to fill <iframe>s for things like video players and like buttons. You can choose how those will be written into your theme (inline, or as URLs) with -inline-image, -no-inline-image, -inline-iframe, and -no-inline-iframe.

    If you don't want to worry about all that, just use the more general -inline or -no-inline to specify inlining behavior, and -tiny to minify everything. Or use that as a starting point and override a couple specifics if you want.

    An Example

    Let's say you are working in a directory accessible by your web server. You have a couple other PCs or VMs or whatever, as well as a mobile device that you want to test out your theme on. In this example, you also have a bunch of scripts and stylesheets to include.

    tumbltool preview -theme theme.html -inline -tiny -include fonts.css colors.css layouts.css a-library.js -tail-include scroll-effects.js -out index.html

    In this example, everything is inline, so you don't have to worry about any file being accessible by your sever other than the preview itself (index.html in this case).

  5. Upload Your Theme

    TumblTool has a bundle function, which basically does whatever inlining and minification you want, without rendering a preview. You can run your theme through this before posting to Tumblr for a bit of free optimization.

    TumblTool's inlining functions can accept URLs too, not just paths. So if you post some stylesheets to Tumblr's static file hosting, and want to pull a JS library from a CDN, just specify the URLs; you don't need to dirty your code with that.

    tumbltool bundle -tiny -inline -include http://static.tumblr.com/my-file http://static.tumblr.com/my-file1 https://some-cdn.com/a-library.js fonts.css -tail-include scroll-effects.js

    In this rather contrived example, fonts.css? will be included inline, and my-file, my-file1 and a-library.js will be included by URL in the <head>, and scroll-effects.js will be included inline at the end of the <body>.
